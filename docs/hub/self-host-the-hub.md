# Self-Host the Hub

You can run your own Phlix Hub instead of relying on a shared/public hub. A
self-hosted hub gives you full control over user accounts, claimed servers, and
relay bandwidth.

Self-hosting the hub is a hub-operator task with its own setup, scaling, and
operational concerns. The full runbook lives in the hub-admin section.

## Where to Start

- [Hub-admin install & first boot](../hub-admin/install.md) — install the hub via
  Docker, docker-compose, Helm, or source, and create the first admin.
- [Hub-admin capacity planning](../hub-admin/capacity-planning.md) — size hardware
  for your expected user and server counts.
- [Hub-admin scaling](../hub-admin/scaling.md) — scale a hub beyond a single instance.
- [Hub-admin monitoring & alerting](../hub-admin/monitoring-alerting.md) — keep the
  hub healthy in production.
